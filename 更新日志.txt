OfficeAI智能助手 更新日志
========================

v2.10 (2026-06-16)
------------------
  修复: 删除 paragraphs.load('paragraphFormat')，解决 v2.9 中所有格式修改（字体/行距/段前段后）全部失效的回归
  原因: Office.js 中对同一对象多次调用 load() 会覆盖而非叠加，导致 p.style 未加载、样式匹配全失败
  说明: paragraphFormat 和 font 同为导航属性，写入不需要显式加载（Office.js 访问时自动追踪）

v2.9 (2026-06-16) [已废弃]
--------------------------
  尝试恢复 paragraphFormat 的 API 写入（段前/段后/对齐/缩进）+ OOXML 双路径
  ⚠️ 引入回归: paragraphs.load('paragraphFormat') 覆盖了 items 加载，所有格式修改失效

v2.8 (2026-06-15) [已废弃]
--------------------------
  将段前/段后/对齐/缩进改为纯 OOXML 注入路径，移除 paragraphFormat API 写入
  ⚠️ 问题: OOXML 替换路径未经验证，在 Word 中实际不生效

v2.7 (2026-06-15)
-----------------
  修复: context.load 错误使用 'items/style/paragraphFormat' 导致样式匹配失败
  回退为 context.load(paragraphs, 'items') 标准方式

v2.6 (2026-06-15)
